The ruler: gaps between species, not cloud size

The pre-flight is the inter-species margin: how cleanly species clusters separate. As a gauge, it forecasts how well both feature-space tasks go (Spearman +0.65 spotting, +0.63 choosing). Effective dimension, how high-dimensional the cloud is, governs neither (+0.22, +0.02). The obvious objection is that separability is just accuracy in disguise; it survives the control. Even after accounting for closed-set accuracy, the margin still keeps ρ=+0.31 (934 clade cells, FWER p<1e−20): real signal, not a restatement of accuracy.

What "detectable" means: two bell curves and the gap between them

Give every observation a novelty score. Known species and novel ones each form a bell curve. Detection quality (AUROC = the chance a random novel outscores a random known) depends on one thing: how far the bells sit apart in their own standard deviations, the separation d′. The identity AUROC = Φ(d′/√2) converts that gap straight into accuracy, exact, no fitting. That is the mechanism, not the evidence. The evidence is the forecast: d′ measured on known species predicts detection of ones held out or absent from training (next slide), and could have failed.

It transfers off the data it was measured on

The margin forecasts per-order difficulty on a continent it was never measured on (+0.59, positive in all six held-out realms), and on species provably absent from the model's training set (the forecast holds at pooled ρ 0.95–0.99; the closed-form identity fits it to Pearson 0.995–0.998, across five backbones and all six realms). Backbone-robust and stable across taxonomic levels (order/family/genus +0.65/+0.61/+0.62).

The closer a known relative sits, the easier the find. Novel lineages with no close relative are the hard case: the standard open-set headline is partly carried by near relatives. The margin forecasts the difficulty at every rung.

So the pre-flight is two-dimensional

Feature-novelty wins species discovery (+20%) and barely moves coverage (+0.9%); geographic coverage wins places (+77%) and barely moves species (+8%). Two independent effects: each axis is a specialist, neither readable off the other. A complete pre-flight needs both.

One geometry, two jobsA gauge and a selector: never one for the other

The gauge (should I target at all?)

  • one number for the whole embedding
  • Φ(d′/√2): how far the score clouds sit apart
  • computed before a single label, from a small reference set
  • verdict: GO / borderline / skip

The selector (which observation next?)

  • one score per candidate point
  • CoreSet: farthest from what's already labelled
  • margin: distance to the nearest known cluster
  • output: the ranked queue
The gauge decides whether to play; the selector decides the move. Same geometry, two different questions. A distance between two clouds (the gauge) never scores a single point (the selector), so keep them apart: the gauge tells you the map is worth working; the selector works it.

One separability, three instrumentsThree ways to read the same gap, and why they aren't interchangeable

kNN-purity selector

  • reads: local, per point, nonparametric
  • gives: a ranking (fit it to get a number)
  • use: pick the backbone, order the queue

Φ(d′/√2) gauge

  • reads: distributional, closed form
  • gives: the AUROC value + a threshold, no fit
  • use: target or skip, set a novelty cutoff

centroid margin mechanism

  • reads: between-class geometry
  • gives: the why (new lineage falls in a gap)
  • use: explain, tie to taxonomic distance
One principle: separability governs discoverability, read three ways. Across 42 embeddings the instruments correlate 0.71–0.88, not 1, so they disagree on real cases. Only the gauge returns the number with no fit, so only it can miscalibrate: bias −0.13 where nothing is detectable, −0.006 where discovery is feasible, while the selector still ranks those cases right.

The ruler again, one level downWhich crown next? Judge by neighbours, not by a block

The pre-flight's ruler said it once: read the local gap between species, not a global summary. The score that ranks which crown to label next lives or dies by the same choice. The standard needle score (Kurinchi-Vendhan & Beery) groups crowns into k-means blocks; labelfirst reads each crown's actual nearest crowns.
Take a rare crown wedged among a common species. k-NN counts who is actually nearest, mostly the common one, and calls it hard. k-means hands the rare species its own cell, where its few crowns are the majority, and calls it easy, ignoring a common crown that is physically nearer but across a border. Hard is right: that crown is found last. So the local read tracks the real discovery order at ρ 0.59 vs 0.40 (four backbone×selector cells, significant); Protium panamense, never found, reads 0.06 hard by k-NN, 0.34 easy by k-means.

labelfirst · the engineA numpy-core selection library

Selection and audit run with zero deep-learning dependencies; torch and pyarrow load lazily, only when you embed or read parquet.
  • Columnar embeddings I/O. key, embedding parquet, read straight from the Arrow value buffer (skips the per-row Python list that triples peak RAM); every row L2-normalized at load, so X @ Xᵀ is cosine.
  • 16 strategies behind one Protocol. select(X, labeled, k) dispatched from a registry; default CoreSet (greedy k-center, cosine). One select_batches call gates, selects, then KMeans-batches for the labeller.
  • The pre-flight is a metric, not a model. kNN leave-one-out separability → GO / borderline / skip; the open-set arm computes AUROC = Φ(d′/√2) from LOO centroid scores, then reports the realized FPR/TPR as a Gaussian-fit check.
  • Every run is auditable. a RunRecord stamps strategy, seed, embedding SHA-256, library and Python versions, timestamp; loaders fail loudly on schema drift.
  • A pre-registered harness. frozen spec, Holm–Bonferroni FWER, a Wilcoxon power-floor that refuses a PASS when seeds can't clear 1/2ⁿ, and a synthetic imbalance self-test where balanced data must show no win.

speciesfirst · the BCI deploymentA governed, conformal curation loop

An auditable loop over real crowns, embed → conformal-tier → Labelbox → WCVP handoff, on a commit-pinned labelfirst for reproducible builds.
  • Conformal tiering with a genus fallback. cross-conformal (K=5) prediction sets split crowns into low-priority (small set, kept as a suggestion, no expert) vs flag-for-review; species→genus→defer, each level held to 1−α coverage; empty sets (out-of-support novelty) rank first.
  • Two-tier trust. only verified (“Done”) labels grow ground truth; in-review labels steer selection but stay out of the pool. merge_labels refuses labels for crowns with no embedding and never overwrites.
  • Labelbox via labelpull. export + flatten; batch = cluster, and clusters below a cohesion floor are dropped, recorded in the push plan, never silently.
  • WCVP handoff to Pl@ntNet. Kew's checklist fixes synonym/rank noise (33% of BCI labels), out-of-range mislabels, and rare-but-weed; deterministic JSON. Crown geometry read from GeoPackages with stdlib sqlite3 + struct — zero geo deps.
  • Leakage-first evaluation. a label-free backtest where the directed selector must beat every random seed, plus a genus-holdout split so a species has no congener prototype to leak from.

The recipe on real crowns roughly doubles skill

On 3 280 BCI crowns: the raw margin picks the best embedding without running the experiment; whitening the survey batch then adds +0.09–0.12 AUROC. End to end, detection skill above chance roughly doubles (0.642 → 0.785). But whitening helps the detector and hurts selection, so: whiten for detection, keep raw for selection.

Why the engine travelsOne pre-flight, every modality a biodiversity lab monitors

Runs today (measured)

  • Tree crowns: measured on real BCI drone crowns; retrain handoff in flight
  • Birdsong, insects, aerial: the same species-gap ruler predicts detection accuracy on one straight line, across six data types (AUROC ≈ 0.32×separability + 0.49, R²=0.96)

Same recipe, not yet run (candidate)

  • UAV tree ID, insect camera-traps, remote-sensing backbone selection
  • same bottleneck (expert time), same frozen-embedding geometry; a dataset manifest away from a test
The binding constraint is the same everywhere: 1–20 min a photo, more pictures than botanists. Detection travels to audio (birdsong AUROC 0.865); the selection-beats-random win is imagery-only so far.

From correlation to cause: a ladder, not a leap

  • Done: a leakage-free backtest where high-priority cells out-discover low-priority ones at equal effort, with an automatic check that stops the run if a held-out crown ever reaches the selector.
  • Next: a candidate-controlled botanist-queue trial, where the labeller acts on exactly what we serve, so there is no compliance gap and no third-party infrastructure.
  • Upside: a randomized bioblitz field trial, once field logistics and approvals align.
